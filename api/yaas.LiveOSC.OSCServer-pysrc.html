 21  RemixNet Module 
 22   
 23  This module contains four classes that have been assembled to facilitate 
 24  remote control of Ableton Live. It's been an interesting experience learning 
 25  Python and has given me a lot of time to think about music and networking 
 26  protocols. I used OSC as it's somewhat of an accepted protocol and at least 
 27  more flexible than MIDI. It's not the quickest protocol in terms of 
 28  pure ops, but it gets the job done.  
 29   
 30  For most uses all you'll need to do is create an OSCServer object, it 
 31  in turn creates an OSCClient and registers a couple default callbacks 
 32  for you to test with. Both OSCClient and OSCServer create their own UDP 
 33  sockets this is settable on initialization and during runtime if you wish 
 34  to change them. 
 35   
 36  Any input or feedback on this code will always be appreciated and I look  
 37  forward to seeing what will come next.

 98          /remix/time -> OSCServer.callbackTime() - Returns time.time() (time in float seconds) 
 99           
100          I chose OSC to deliver messages out of necessity, my opinion of OSC at this 
101          point is that its addressing system is heavyweight although the idea is  
102          a reasonable one. But taking into consideration the ratio of address:data 
103          it becomes somewhat unreasonable unless you take the route of making unreadable 
104          addresses. As an example I offer the following address, 
105   
106          /ableton/track/1/volume/set float(0.98) 
107   
108          To set a single 4 byte float value, we need to use a 27 byte string to get 
109          it routed to the correct area, and even then we need to make an O(N) comparison 
110          on the address since we don't have the luxury of a switch statement in Python. 
111           
112          If you're trying to interact with devices in near-realtime the number of ops 
113          wasted on just getting things to the right place can take the wind out of 
114          your sails. 
115           
116          But basically for this project to be accepted or useful to anyone it was  
117          important to me that we provide a method of accessing that other tools 
118          could use without having to introduce a new linewire protocol. 
119           
120          It should be noted that performance even with the added ops isn't that bad. 
121          On my dualcore system I was able to process about 1380 OSC callbacks per
